**DECISION_105 IMPLEMENTATION - Automated Silent-Failure Detection**

The Nexus has institutionalized 5 silent-failure patterns. DECISION_105 is canon. Designer approved at 95%. Now we build the tools to detect them automatically.

**THE FIVE PATTERNS TO DETECT:**

1. **Empty catches or ignored exceptions**
   - Search: `catch(Exception)` with no `throw`
   
2. **Default-success returns**
   - Search: `return true` inside catch blocks
   
3. **Async void handlers**
   - Search: `async void` method signatures
   
4. **Logging without failing**
   - Search: `LogWarning`/`LogError` followed by `return`
   
5. **Skipped actions leaving invalid state**
   - Search: Conditional blocks that log then return success

**YOUR MISSION:**

### 1. AST-grep Rules (Immediate)
**File**: `STR4TEG15T/tools/ast-grep-rules/silent-failures.yml`

Create rules for each pattern:
```yaml
rules:
  - id: empty-catch
    pattern: |
      catch ($$EX: Exception) {
        $$BODY
      }
    condition: $$BODY not contains "throw"
    
  - id: default-success-return
    pattern: |
      catch ($$EX) {
        $$LOG
        return true;
      }
    
  - id: async-void
    pattern: |
      async void $$METHOD
```

### 2. Pre-commit Hook
**File**: `STR4TEG15T/tools/pre-commit/tychon-guard.sh`

Run AST-grep on commit:
```bash
#!/bin/bash
ast-grep scan --config silent-failures.yml
if [ $? -ne 0 ]; then
    echo "TYCHON GUARD: Silent failure patterns detected. Commit blocked."
    exit 1
fi
```
